## 8. نظام الإعلانات، الترويج والتحليلات (Advertisements & Analytics)

يوفر هذا النظام وسيلة ديناميكية لإدارة المحتوى الترويجي داخل التطبيق، مع دعم كامل للـ (Deep Linking) وتتبع الأداء اللحظي.

### 8.1 آلية عمل التوجيه (Targeting Logic)
عند جلب الإعلانات، يحتوي كل كائن على `target_type` و `target_id`. يجب على مطور الجوال التعامل معها كالتالي:
- **`service`**: يجب فتح صفحة تفاصيل الخدمة التي تحمل المعرف `target_id`.
- **`category`**: يجب فتح قسم الخدمات المفلتر بالقسم الذي يحمل المعرف `target_id`.
- **`external`**: يجب فتح الرابط الموجود في `external_link` في متصفح خارجي.
- **`none`**: الإعلان للعرض فقط ولا يستجيب للنقر.

### 8.2 جلب الإعلانات النشطة `GET /api/advertisements` (عام/Auth)
**الوصف:** جلب البنرات والعروض الموجهة للمستخدم بناءً على نوع حسابه.
- **المعاملات (Query Params):**
  - `user_type`: (`all`, `client`, `provider`) لفلترة الإعلانات المخصصة لكل فئة.
- **النتيجة:** مصفوفة من الإعلانات تشمل الروابط، الأنواع، وبيانات التوجيه.

### 8.3 تتبع الظهور (View Tracking) `POST /api/advertisements/{id}/view` (عام/Auth)
**الوصف:** يجب استدعاء هذا المسار فور ظهور الإعلان على شاشة المستخدم (Impression).
- **الهدف:** زيادة عداد المشاهدات في لوحة تحكم الإدارة لتقييم مدى وصول الإعلان.

### 8.4 تتبع النقرات (Click Tracking) `POST /api/advertisements/{id}/click` (عام/Auth)
**الوصف:** يجب استدعاء هذا المسار عندما يقوم المستخدم بالنقر على الإعلان.
- **الهدف:** قياس نسبة التفاعل (CTR) وتحديد الإعلانات الأكثر جذباً للمستخدمين.

### 8.5 هيكل بيانات الإعلان (Data Schema)
```json
{
    "id": 1,
    "title": "عرض العيد",
    "image_url": "http://.../storage/advertisements/image.jpg",
    "type": "carousel",
    "target_type": "service",
    "target_id": 15,
    "external_link": null,
    "metrics": {
        "views": 150,
        "clicks": 45
    }
}
```
> [!TIP]  
> ينصح بـ (Cache) للصور في التطبيق لضمان سرعة التصفح، مع ضرورة إرسال طلبات التتبع (Views/Clicks) في الخلفية لضمان دقة التقارير الإحصائية للمدير.
